# Category 7: Tutorial Images / Diagrams Mismatch Actual UI

**Difficulty estimate**: 2-3 (Easy to Medium)

## Detection Signals

- Tutorial text describes an action but the embedded diagram shows a different board state than the actual game UI
- Tutorial shows colors, piece shapes, or board layouts that differ from the live game
- Screenshot comparison: tutorial image shows round pieces, game shows square tokens
- Tutorial text says "click the blue stone" but the actual game uses red/green coloring

## Root Causes

1. Tutorial was written when the game had a different visual theme; game was later re-themed
2. Tutorial uses static image files that weren't updated when the renderer changed
3. Tutorial board state is generated by `renderTutorialBoard` which uses different colors/sizes than the actual game renderer
4. Stacks (Tak) tutorial shows generic board images, not the actual 3D react-three-fiber scene

## Known Fix Patterns

### Pattern A: Update `renderTutorialBoard` to match actual renderer colors
Find the tutorial's `renderTutorialBoard` function in `App.js` and compare the colors/sizes it uses against the live `renderBoard`/SVG function. Sync them.

### Pattern B: Replace static image files with screenshots of the real game
If the tutorial uses `<img src="tutorial-step-1.png">` style assets, re-capture them from the live game at the correct state.

### Pattern C: Use the shared `GridTutorialBoard` component
For grid games (Go, Fives, Flips, Stones), the `TutorialBoards.jsx` `GridTutorialBoard` component uses the same color constants as the main game. Use this component rather than custom SVG.

## Known Failure Cases

- Stacks (Tak) 3D board cannot be embedded in a tutorial iframe. Use simplified 2D top-down SVG representation instead of trying to show the actual 3D scene.
- If tutorial board states are defined in JSON with hex color codes, they can drift from the live game's CSS variables. Prefer using the same JS constants for both.

## Known Failure Cases (added 2026-03-28)

- **Text references wrong UI location**: About page said "bottom-right corner" for bug report button, but FAB is positioned `left: 20px` (bottom-left). Any text that references the position of a UI element must match the actual CSS positioning.
- **Directional text audits**: Search for spatial references like "bottom-right", "top-left", "below", "above" in all user-visible text. Cross-reference each against the actual CSS `position`, `top/bottom/left/right` values.

## Verification

1. Side-by-side comparison: capture a screenshot of the live game at a state shown in the tutorial, then visually compare it to the tutorial's diagram. Colors, piece shapes, and board grid should match.
2. Text-position audit: grep for directional terms ("bottom-right", "bottom-left", "top-right", "top-left") in all JSX/HTML and verify they match the actual layout.
